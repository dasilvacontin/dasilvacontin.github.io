Descripción

El objetivo principal de este curso es aprender a crear juegos multijugador online para cualquier plataforma, sin importar el lenguaje o framework utilizado. Esto significa que, aunque los ejercicios y proyecto final serán realizados con un cliente HTML5 y un servidor Node.js, el conocimiento adquirido sobre diseño de juegos multijugador online y algoritmos varios es aplicable a cualquier lenguaje de programación y plataforma – no se utiliza un framework o plugin que te abstrae todo el código de networking, si no que se conocerán los diferentes métodos y algoritmos disponibles, sus trade-offs, y cómo implantarlos.

El stack de desarrollo HTML5 + Node.js + Socket.io es ágil, cross-platform, y no necesita instalación por parte del cliente – perfecto para iterar, probar diferentes técnicas o algoritmos, prototipar, gamejams, MVP, etc.

Temario

  • Creación de videojuegos en el navegador (4h + 2h no presenciales):
    • Introducción a la programación con JavaScript
    • Introducción al renderizado mediante HTML5 Canvas
    • Uso de Spritesheets
    • Interficies de input: keyboard y mouse
    • Arquitectura y funcionamiento de un juego
    • Elaboración de un juego simple
  • Introducción al desarrollo de juegos multijugador online (4h + 2h no presenciales):
    • Funcionamiento de internet y direccionamiento IP
    • Funcionamiento de sockets y puertos
    • TCP y UDP
    • Introducción a Node.js y npm
    • Introducción a Express.js, y el funcionamiento del protocolo HTTP y de los servidores web
    • Introducción a Socket.io
    • Arquitectura simple para el funcionamiento de un juego multijugador online
    • Introducción a los aspectos a considerar al diseñar la arquitectura y funcionamiento de un juego multijugador online
    • Elaboración de un juego multijugador online de arquitectura simple
    • Despliegue del sistema a producción continuo (continuous deployment) mediante GitHub y Heroku
  • Algoritmos y diseño del aspecto red de un juego (2h + 4h no presenciales)

    Se explicarán diferentes algoritmos y técnicas usadas en juegos multijugador online para resolver problemas inherentes que estos presentan, sus pros, contras, con la ayuda de visualizaciones interactivas. Se comentará su implementación y se proveerá de código de ejemplo. El alumno pondrá en práctica el conocimiento ya sea mediante experimentos por cuenta propia o por su uso en el proyecto final.

    • Client-side authority frente a Server-side authority
    • Client-side prediction. Dead reckoning
    • Server reconciliation
    • Estado lógico frente a estado visual. Interpolación.
    • Sincronización sólo mediante intercambio de inputs. Lockstep y simulaciones deterministas con rollback.
    • Load scaling mediante uniform grids.
    • Técnicas de optimización de ancho de banda. Propiedades deterministas, Server Dead Reckoning, y Hierarchical Replication.
    • Partición del mundo mediante un servidor distribuido y funcionamiento de las fronteras entre particiones.
  • Funcionalidades comunes en juegos multijugador online (40min + 2h no presenciales)
    • IA / NPC
    • Chat
    • Introducción a SQL
    • Login y Registro. Ranking.
    • Persistencia del estado del juego.
  • Proyecto final: desarrollo de un juego multijugador online (9h 20min + 6h no presenciales)

    El alumno pone en práctica los conocimientos adquiridos mediante su integración en un juego multijugador online. Con ayuda del profesor, escoge qué técnicas y algoritmos utilizar en función del tipo de juego que desea desarrollar; o en función de las técnicas y algoritmos que quiere poner en práctica, encuentra un tipo de juego para el que éstos sean una buena elección. El profesor resolverá dudas, problemas, y dará indicaciones de cómo implementar (o directamente implementará para dar ejemplo) aquellas funcionalidades que se le pida.

    Antes de finalizar el curso se realizará una presentación de los juegos desarrollados, en la que cada alumno presentará su juego, explicará las decisiones de diseño tomadas, algoritmos utilizados, y se procederá al playtest del juego, en el que todos los alumnos probarán el juego simultáneamente a través de su navegador.

    (Opcional) Una semana más tarde se realiza otro playtest, en el que los alumnos y el profesor quedan de forma online (a través de un chat) para probar los proyectos de nuevo. Se realiza para motivar a los alumnos a la continuación de su proyecto, consolidar conocimientos adquiridos, ir más allá. El profesor estará disponible toda la semana para ser consultado respecto a dudas.
